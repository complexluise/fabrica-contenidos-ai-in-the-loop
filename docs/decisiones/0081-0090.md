## D-081 — El frontend se vuelve Svelte de verdad: mutación directa, remount por key y verdad compartida
**Estado:** Vigente · **Fecha:** 2026-06-12 · **Sprint:** 6.37

- **Contexto:** La revisión de prácticas (post-D-080) encontró que el código USA Svelte 5
  (runes correctos, store global idiomático) pero ESCRIBE como React en varios frentes:
  34 spread-clones (`x = {...x, [k]: v}`) donde `$state` ya es profundamente reactivo;
  un `$effect` que reseteaba 10 variables a mano al cambiar de proyecto (bug de estado
  fantasma esperando a que alguien olvide la #11); cero `components/` — la lighttable
  implementada DOS veces (Casting/Encuadres), el header de generación TRES, el ciclo de
  vida de jobs OCHO (la música estuvo siempre rota porque UNA copia divergió: la lección
  de D-080), la reconciliación de picks-desde-disco DOS; `fetch()` crudo saltándose la
  capa `api.js`; y sin routing — F5 devolvía al Inicio. **Principio situado** (herramienta
  local, un operador): duplicar PÍXELES es barato y se tolera; duplicar VERDAD (lógica,
  modelos, ciclos de vida) es lo que produce los bugs de des-sincronización que venimos
  matando. No corresponde aquí: design system, TypeScript, test framework de UI, router
  externo.

- **Decisión:**
  1. **Mutación directa de `$state`**: `picks[id] = i`, `log.push(l)`, `delete picks[id]`.
     El spread-clone muere en todo el árbol (era el hábito React; en Svelte 5 los proxies
     hacen el trabajo).
  2. **`{#key studio.slug}`** alrededor de las vistas de proyecto en App: cambiar de
     proyecto REMONTA la vista — muere todo reset manual de estado (`onMount(load)` y listo).
  3. **`lib/jobs.svelte.js :: jobState()`**: UNA implementación del ciclo
     busy/progress/err/log + runJob. Ocho copias mueren; divergir vuelve a ser imposible.
  4. **`lib/picks.js :: picksFromDisk()`**: la reconciliación selección-en-disco → índice,
     una sola vez.
  5. **`components/`** SOLO donde hay verdad o estructura compartida real: `LightTable`
     (la mesa de luz con pick/descartar/badges — x2), `GenerateBar` + `BackendToggle`
     (header de generación con costo visible — x3), `ViewHeader`, `WarnStrip`, `WarnBanner`,
     `Progress`, `SaveBar`. Snippets/children de Svelte 5 para el contenido variable.
     El Storyboard NO se parte: no tiene duplicación (partir por partir es el culto que
     este ADR rechaza).
  6. **Routing por hash** (`#/<slug>/<tab>`, ~25 líneas en el store): F5 y el botón atrás
     conservan dónde estabas; una URL de pestaña se puede compartir. Sin router externo.
  7. `fetch` crudo → `api.get` (una sola capa con `humanError`).

- **Consecuencias:** menos líneas, una sola fuente para cada verdad de UI, y el patrón
  nuevo queda demostrado en el código (la próxima vista se escribe componiendo, no
  copiando). El CSS duplicado de píxeles que no viajó a componentes se queda donde está,
  a propósito.

- **Cambios:** `app/src/lib/jobs.svelte.js` y `lib/picks.js` (nuevos), `lib/studio.svelte.js`
  (hash routing), `components/*` (nuevos), `App.svelte` ({#key} + hash), `Casting.svelte` y
  `Encuadres.svelte` (reescritos sobre las piezas), `Animatic/Produccion/Importar/Storyboard/
  Ajustes` (mutación directa + jobState + adopción).

- **Addendum (misma fecha):** segunda pasada de adopción — `StageNode` (el nodo del bucle,
  duplicado entre App e Inicio) y `JobLog` (el registro en vivo, x2); `ViewHeader`/`WarnStrip`
  adoptados en TODAS las vistas (WarnStrip había quedado creado sin usar — lo peor de ambos
  mundos); utilidades comunes (`.muted/.empty/.note/.group`) a `app.css`. De paso cayeron
  **4 `goTo("elegir")` muertos dentro del Storyboard** (modo lectura + barra de firma:
  "Siguiente: Elegir" daba pantalla en blanco — D-080 no los vio). Verificación honesta del
  conteo: el neto de líneas queda casi plano (~3.820) porque los componentes nuevos compensan
  lo borrado — lo que bajó de verdad es vistas+App (3.177 → 2.982 con MÁS funcionalidad) y a
  cero las copias de cada verdad. Un detector de CSS muerto confirmó que el resto del CSS es
  diseño vivo: de aquí hacia abajo sería compresión por deporte.
